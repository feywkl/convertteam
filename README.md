# 📊 Отчёты Яндекс.Директ + Метрика → Google Sheets через Telegram-бота

Этот бот помогает быстро получать отчёты по рекламе и Метрике прямо в Google Таблицу. Вы просто отправляете команду в Telegram, а бот сам собирает данные, считает показатели и записывает всё в нужный лист.

## Коротко о работе

1. Откройте Telegram и отправьте команду.
2. Укажите логин клиента и период отчёта.
3. Бот соберёт данные из Яндекс.Директа и Яндекс.Метрики.
4. Отчёт появится в Google Таблице.

Пример команды:

```text
/report client-example 01.01.2026-02.01.2026
```

## Как пользоваться

### Что писать в команде

Формат команды такой:

```text
/report ЛОГИН_КЛИЕНТА ПЕРИОД
```

Примеры:

```text
/report client-example 2026-05-06
```

```text
/report client-example 2026-05
```

```text
/report client-example 01.01.2026-02.01.2026
```

### Как указать период

Можно написать период в одном из трёх вариантов:

- одна дата: `2026-05-06` или `06.05.2026`
- месяц: `2026-05` или `month:2026-05`
- диапазон дат: `01.01.2026-02.01.2026` или `2026-01-01:2026-01-31`

### Что вы получите

Бот выгружает отчёт в таблицу с такими данными:

- дата
- расход
- показы
- клики
- CPC
- CTR
- конверсии
- CPA
- CR

В конце добавляется строка `ИТОГ`.

### Если бот не отвечает

Проверьте три вещи:

- вы пишете команду в правильном формате
- ваш логин разрешён для бота
- отчётный период указан без ошибки

### Если отчёт пустой

Обычно это значит, что за выбранный период нет данных в Директе или указан не тот клиент.

### Если данные не попадают в таблицу

Проверьте, что у сервисного аккаунта есть доступ на редактирование Google Таблицы.

## Для администратора

Ниже — техническая часть для тех, кто настраивает бота и деплоит его на сервер.

### Что нужно для работы

- токен Telegram-бота
- токен Яндекс.Директа
- токен Яндекс.Метрики
- ссылка на Google Таблицу
- файл `credentials.json` для доступа к Google Sheets

### Как бот понимает клиента

Если у вас один общий счётчик Метрики, отдельный `clients.json` не нужен: бот может работать просто по логину клиента.

Если у разных клиентов разные счётчики Метрики, можно оставить `clients.json` как справочник с настройками клиентов.

### Ограничение доступа

Бот отвечает только разрешённым пользователям. Список разрешённых chat_id можно задать через переменную окружения `TELEGRAM_ALLOWED_CHAT_IDS` или через локальный файл `telegram_allowlist.json`.

### Полезные команды для админа

- `/myid` — показать свой `chat_id`
- `/addadmin <chat_id|@ник>` — добавить администратора
- `/removeadmin <chat_id>` — удалить администратора
- `/admins` — показать список администраторов

### Локальный запуск

```bash
pip install -r requirements.txt
python generate_report.py
```

Для запуска бота:

```bash
python telegram_bot.py
```

### Запуск на сервере

Обычно бот запускают через `systemd` как постоянный сервис. Если нужно, можно дополнительно настроить proxy и расписание для автозапуска отчётов.

### Где смотреть настройку

- `telegram_bot.py` — логика Telegram-бота
- `report_core.py` — сбор и запись отчёта
- `config.py` — загрузка настроек из окружения
