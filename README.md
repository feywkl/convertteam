# 📊 Авто-отчеты: Яндекс.Директ + Метрика → Google Таблицы

## 👨‍💻 Инструкция для менеджеров (Как работать с системой)

Вам не нужно устанавливать программы или разбираться в коде. Всё управление происходит прямо здесь.

### 1. Как добавить нового клиента
Чтобы система начала собирать данные по новому проекту, нужно просто добавить его в список.

1. Нажмите на файл `clients.json` в списке файлов выше (или [нажмите сюда](clients.json)).
2. Справа вверху над кодом нажмите на иконку **карандаша ✏️** (Редактировать файл).
3. Скопируйте блок предыдущего клиента, поставьте после него запятую `,` и вставьте копию ниже.
4. Отредактируйте данные:
   * **Ключ** (`"client_1"`) — назовите как угодно латиницей, это для системы.
   * `direct_login` — логин клиента в Яндекс.Директе.
   * `metrika_counter` — номер счетчика Яндекс.Метрики.
   * `worksheet_name` — как должна называться вкладка в Google Таблице. Скрипт **создаст её сам**.
5. Нажмите зеленую кнопку **"Commit changes..."** в правом верхнем углу (и еще раз подтвердите).
*Готово! На следующее утро вкладка появится в таблице.*

### 2. Как запустить обновление таблицы прямо сейчас (вручную)
Если вы добавили клиента и не хотите ждать утра, или вам нужно выгрузить статистику за прошлый месяц:

1. Перейдите во вкладку **Actions** (в самом верху страницы GitHub).
2. В левом меню нажмите на скрипт (например, **Daily Report**).
3. Справа нажмите на серую кнопку **Run workflow**.
4. Появятся два поля: `Date from` и `Date to`.
   * *Опционально:* Впишите даты в формате `2026-04-01` (год-месяц-день).
   * Если оставить поля пустыми, скрипт обновит данные только за вчерашний день.
5. Нажмите зеленую кнопку **Run workflow**. Подождите пару минут, и Гугл Таблица обновится!

## ⚙️ Инструкция для администратора (Смена доступов / Ссылок)

Если вы завели новую таблицу или обновили токены Яндекса, вот где их поменять:

**Путь настройки:** Вкладка **Settings** (наверху) → в левом меню раскрыть **Secrets and variables** → нажать **Actions**.

Там лежат 4 секретных ключа (в разделе *Repository secrets*):
* `GOOGLE_SHEET_URL` — ссылка на вашу Google Таблицу (обязательно дайте доступ редактора сервисному email-адресу!).
* `DIRECT_TOKEN` — токен от Яндекс.Директа.
* `METRIKA_TOKEN` — токен от Яндекс.Метрики.
* `GOOGLE_CREDENTIALS` — служебный код Google.

**Как изменить:** нажмите на иконку карандаша ✏️ рядом с нужным секретом, вставьте новый текст и нажмите зеленую кнопку **Update secret**.

## 🛠 Памятка по таблице (Какие колонки выгружаются)
Скрипт автоматически рисует жирные рамки, объединяет данные по месяцам и считает "ИТОГ". Столбцы:
1. Дата (или ИТОГ)
2. Расход, руб
3. Количество показов
4. Клики
5. CPC (Цена клика)
6. CTR 
7. Конверсии
8. CPA (Стоимость конверсии)
9. CR (Конверсия сайта)

## 🧰 Как запускать локально (для технического сотрудника)

Если нужно отладить или запустить скрипт вручную на машине разработчика/администратора:

1. Склонируйте репозиторий и установите зависимости (рекомендуется виртуальное окружение):

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2. Создайте `credentials.json` с содержимым из секрета `GOOGLE_CREDENTIALS` (или используйте `gspread` напрямую).

3. Экспортируйте секреты в окружение и запустите за нужный период:

```bash
export DIRECT_TOKEN="..."
export METRIKA_TOKEN="..."
export GOOGLE_SHEET_URL="https://docs.google.com/...."
export GOOGLE_CREDENTIALS_FILE=credentials.json
DATE_FROM=2026-04-01 DATE_TO=2026-04-30 python3 generate_report.py
```

4. Логи запуска покажут какие листы обновлялись и сколько строк было выгружено.

## 🗂 Формат `clients.json` и быстрый пример

Файл находится в корне репозитория. Простой и понятный формат — ключи по клиентам с их настройками. Пример:

```json
{
   "metall-cvt": {
      "direct_login": "metall-cvt",
      "metrika_counter": "53749281",
      "worksheet_name": "metall-cvt"
   }
}
```

Правила:
- Используйте латиницу для ключей (они же отображаются в логах).
- `worksheet_name` — именно так будет называться вкладка в Google Таблице.
- Если надо — добавляйте `metrika_goal_id` (опционально).

После правки `clients.json` — сохраните (Commit) и запушьте на GitHub: система подхватит изменения при следующем запуске.

## ⚠️ Частые ошибки и как их решать

- "OAuth-токен не указан" (Директ): проверьте, что секрет `DIRECT_TOKEN` задан в *Repository secrets*.
- "Ошибка доступа к таблице": убедитесь, что email сервисного аккаунта имеет доступ к таблице (редактор).
- Пустые дни в таблице: Яндекс.Директ не возвращает строки с нулевой активностью — это нормально.
- Ошибка формата `clients.json`: проверьте JSON валидность; используйте `json.tool` для проверки:

```bash
python -m json.tool clients.json
```
