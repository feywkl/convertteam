# Авто-обновление отчёта (Яндекс.Директ + Метрика → Google Sheets)

Этот репозиторий обновляет Google Таблицу с рекламной статистикой: забирает данные из API Яндекс.Директ и Яндекс.Метрика, считает производные метрики и формирует таблицу с помесячными итогами.

## Как это работает

1. Скрипт `generate_report.py` читает список клиентов из `clients.json`.
2. Для каждого клиента:
   - Запрашивает отчёт Яндекс.Директ (Reports API v5) за период `DATE_FROM` → `DATE_TO`.
   - Запрашивает статистику Яндекс.Метрики (Stat API v1) за тот же период.
3. Данные агрегируются **по дням**.
4. Таблица формируется блоками по месяцам:
   - Шапка (жирным)
   - Строки по дням
   - Строка **ИТОГ** (жирным)
   - 2 пустые строки-разделителя

Важно: текущая реализация пишет данные в **первый лист** таблицы (`sheet1`) и очищает его при перезаписи.

## Какие колонки формируются

Сейчас выгружаются 9 колонок:

1. Дата (формат `DD.MM.YYYY`)
2. Расход, руб.
3. Количество показов
4. Кол-во кликов
5. CPC = `cost / clicks`
6. CTR = `clicks / impressions * 100%`
7. Кол-во конверсий
8. Стоимость конверсии (CPA) = `cost / conversions`
9. CR = `conversions / clicks * 100%`

Примечание:
- Расход/ставки берутся из Яндекс.Директ.
- Конверсии берутся из поля `Conversions` отчёта Директа.
- Метрика сейчас используется для получения сессий (данные подтягиваются по дням), но **в текущем формате таблицы сессии не выгружаются**.

## Период отчёта (как менять даты)

По умолчанию скрипт работает за **вчерашний день**.

Можно переопределить период через переменные окружения:

- `DATE_FROM` — дата начала в формате `YYYY-MM-DD`
- `DATE_TO` — дата конца в формате `YYYY-MM-DD`

Примеры:

```bash
DATE_FROM=2026-04-16 DATE_TO=2026-04-16 python3 generate_report.py
DATE_FROM=2026-05-01 DATE_TO=2026-05-31 python3 generate_report.py
```

## Список клиентов (clients.json)

Файл `clients.json` — это список клиентов, по которым собирается статистика.

Пример:

```json
{
  "client_1": {
    "direct_login": "metall-cvt",
    "metrika_counter": "53749281",
    "worksheet_name": "Металл ЦВТ"
  },
  "client_2": {
    "direct_login": "other-login",
    "metrika_counter": "11111111",
    "worksheet_name": "Другой клиент"
  }
}
```

Поля:
- Ключ (`"client_1"`) — как будет отображаться в логах скрипта.
- `direct_login` — логин клиента в Яндекс.Директ (используется как `Client-Login`).
- `metrika_counter` — ID счётчика Метрики.
- `worksheet_name` — (опционально) название вкладки (листа) в Google Таблице. Если лист не существует, он будет создан автоматически. Если не указать, листом будет имя ключа клиента.


## Автозапуск в GitHub Actions

- Запускается по расписанию (cron) и вручную.
- Устанавливает зависимости.
- Создаёт `credentials.json` из секрета `GOOGLE_CREDENTIALS`.
- Запускает `python generate_report.py`.

### Ручной запуск с датами

В Actions → Workflow → **Run workflow** можно (опционально) указать `date_from` и `date_to`.
Если оставить пустыми — скрипт возьмёт «вчера».

## Как менять секреты в GitHub

Путь: **Repository → Settings → Secrets and variables → Actions → Secrets**.

Секреты, которые используются:
- `DIRECT_TOKEN` — OAuth-токен Яндекс.Директ.
- `METRIKA_TOKEN` — OAuth-токен Яндекс.Метрика.
- `GOOGLE_SHEET_URL` — URL Google Таблицы.
- `GOOGLE_CREDENTIALS` — содержимое JSON service account (вставляется целиком).

Как обновить:
1. Откройте нужный секрет.
2. Нажмите **Update secret**.
3. Вставьте новое значение.
4. Сохраните.

Важно по `GOOGLE_CREDENTIALS`:
- Вставляйте **весь JSON** как текст (не base64).
- Убедитесь, что в Google Таблице выдан доступ email’у сервисного аккаунта.

## Частые проблемы

- `OAuth-токен не указан` (Директ): не передан `DIRECT_TOKEN` или токен протух.
- Ошибка доступа к таблице: сервисному аккаунту не выдали доступ к Google Sheet.
- Данные «пропали»: скрипт очищает `sheet1` при перезаписи — держите этот лист только под отчёт.
